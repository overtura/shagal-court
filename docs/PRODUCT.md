# 제품

## 문제와 약속

사용자는 억울하거나 황당한 일을 짧게 정리하고 가볍게 웃을 수 있는 판결을 원한다. 샤갈 재판소는 법적 결론 대신 설명 가능한 밈 판결을 만들며, 원문을 서버 AI로 보내지 않는다.

## 사용자 흐름

1. 180자 이하 사건 진술을 입력한다.
2. 브라우저가 WebGPU, WASM, 결정론적 대체 방식 순으로 분석한다.
3. 분류와 7개 축에서 순수 판결 엔진이 네 종류의 밈 판결을 만든다.
4. 로컬 사건부와 로컬 현장 배심원 투표는 그 브라우저에만 저장된다.
5. 공개하려면 별도 패널에서 공개 문장을 수정하고 90일 보관·전역 투표에 명시적으로 동의한다.
6. Worker가 개인정보·위협·마크업을 검사하고 분석값에서 판결을 다시 계산한다.
7. 검색에 노출되지 않는 `/case/:slug` 고유 URL에서 읽기, 전역 투표 수정, 신고가 가능하다.
8. 신고 기준치, 관리자 숨김, 삭제, 만료는 모두 동일한 일반 404로 외부에 보인다.

## 판결

- `shagal-guilty`: 상대 책임과 황당함이 충분하다.
- `mitigated`: 황당하지만 사정참작이 있다.
- `insufficient`: confidence가 낮아 심리를 보류한다.
- `shagal-not-guilty`: 오해·맥락 가능성이 더 크다.

금지 결과는 법 조항, 형량, 벌금, 고소 가능성, 범죄 성립, 범죄자 단정, 의료·법률 진단, 보복 권고, 혐오·차별 증폭이다.

## 비범위

계정, OAuth, 댓글, 파일 업로드, 실시간 피드, 공개 목록, 검색, 순위, 주기적 조회, 분석 도구, 외부 유료 API, 서버 AI는 만들지 않는다.

## 성공 기준

- 네트워크가 없어도 로컬 판결이 나온다.
- 공개 동의 전 API 요청이 없다.
- 서버가 클라이언트 판결 문자열을 신뢰하지 않는다.
- 같은 기기의 투표 수정은 총합을 이동하고 중복 증가하지 않는다.
- 360px, 768px, 1440px와 키보드, reduced motion에서 핵심 흐름이 작동한다.
